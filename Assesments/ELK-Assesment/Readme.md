# Centralized Logging Setup using ELK Stack

**Main Repository:** [https://github.com/nihar-landge/python-app-elk-logging](https://github.com/nihar-landge/python-app-elk-logging)

A centralized logging pipeline built with Docker Compose.
Logs are generated by a Python app, shipped via Filebeat, parsed in Logstash, stored in Elasticsearch, and visualized in Kibana.

---

## Architecture

```
sample-app (Python)
     │
     │  writes JSON logs to /logs/app.log
     ▼
  Filebeat
     │
     │  ships to Logstash via port 5044
     ▼
  Logstash
     │
     │  parses JSON logs
     ▼
Elasticsearch
     │
     │  stores in index: app-logs-YYYY.MM.dd
     ▼
  Kibana
     │
     └─ Discover → view and filter logs
     └─ Dashboard → visualize log data
```

### Run the stack

```bash
git clone https://github.com/nihar-landge/python-app-elk-logging.git
cd python-app-elk-logging
docker compose up --build -d
```

Single command to start all 5 services.

### Check if everything is running

```bash
docker compose ps
```

### Access Kibana

Open browser: **http://localhost:5601** ,
if using the VM **<public-ip:5601>**

Wait ~2-3 minutes for all services to start.

---

## Kibana Setup

### 1. Create Index Pattern
1. Go to **Stack Management → Index Patterns**
2. Click **Create index pattern**
3. Name: `app-logs-*`
4. Time field: `@timestamp`
5. Click **Save**

### 2. View Logs in Discover
1. Go to **Kibana → Discover**
2. Select `app-logs-*`
3. Logs will appear in real time

## Example Filters (KQL)

```kql
# Only ERROR logs
level: "ERROR"

# WARNING and ERROR logs
level: "WARNING" or level: "ERROR"

# Only INFO logs
level: "INFO"
```

## Dashboard

Dashboard contains two panels:

**1. Log Level Distribution (Pie Chart)**
Shows the percentage breakdown of log levels:
- INFO: 44.55%
- WARNING: 27.68%
- ERROR: 14.65%
- DEBUG: 13.13%

**2. The Heartbeat (Area Chart)**
Shows count of log records over time (per 30 seconds).
X-axis: `@timestamp` | Y-axis: Count of records
